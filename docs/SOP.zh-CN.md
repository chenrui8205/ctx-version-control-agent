# Context VCS 使用手册（团队版）

一句话说明这个工具：**团队的"工作记忆库"。** 每个人做完一段工作，用几句话推送(push)一次；
系统自动整理成条目、去掉重复、合并更新；如果两个人的说法互相矛盾，它不会悄悄覆盖，而是
交给管理员裁决。任何人（以及任何人的 AI 助手）随时可以看到团队最新的工作状态。

---

## 一、注册（只做一次，约 2 分钟）

1. 打开团队网址（管理员会发给你），点击「没有账号？注册」。
2. 填写邮箱、密码、昵称，以及管理员发给你的**邀请码**。
3. 注册成功后会看到欢迎页，上面有两条命令——复制到你电脑的终端里执行：

```bash
pipx install "git+<仓库地址>#subdirectory=cli"   # 第一条：安装命令行工具
ctxvcs login --api https://<团队域名>/api --web https://<团队域名>
# 第二条：登录（输入你刚才注册的邮箱和密码）
```

完成。不需要复制任何令牌或 ID——登录会自动配置好一切。

> 注意：在新设备上重新登录会使旧设备的登录失效（每人同时只有一份有效凭证），
> 重新 `ctxvcs login` 即可。

## 二、日常使用：推送你的工作（每次 1-2 分钟）

每段工作结束后，在终端运行：

```bash
ctxvcs push
```

会打开一个编辑器，用几句话（要点式即可）写下这次工作：
- **决定**了什么（以及否决了什么方案、为什么）
- **发现/查明**了什么
- **改动**了什么
- **提出**或**解决**了什么问题；接下来要做什么（谁负责）

保存退出后，系统自动整理并与团队现有认知合并，然后告诉你结果：

- `new` 新增 · `update` 更新了旧条目 · `dropped` 团队已经知道了（自动去重）
- `CONFLICT` 与现有记录矛盾 → 终端会给出链接，等管理员在网页上裁决，**不需要你做任何事**

**一个重要的写法习惯：** 改动要写成"改动"，不要写成"事实陈述"。
- ✅ 「把超时从 60 秒改成 30 秒（压测后决定）」→ 干净地更新旧记录
- ❌ 「超时是 30 秒」→ 系统无法判断这是更新还是矛盾，可能触发人工审核

也可以不打开编辑器：`ctxvcs push --file 笔记.md` 或 `echo "..." | ctxvcs push --stdin`。

## 三、日常使用：查看团队状态

**网页端**（推荐，全中文界面）：打开团队网址登录。
- **进行中**：当前所有待办和未决问题——"我该接着做什么"看这里
- **工作日志**：每次推送的记录，最新在前——"最近发生了什么"看这里
- 每个主题页有 **Facts 表**（当前的准确数值/结论）；点 **「溯源」** 按钮可以看到
  任何一条信息是谁记录的、什么时候、有没有被质疑过、谁裁决的——不确定该不该信一条
  信息时就点它。

**命令行**：
```bash
ctxvcs threads          # 进行中
ctxvcs journal          # 最近的工作日志
ctxvcs page <主题名>     # 某个主题的详情
ctxvcs search <关键词>   # 搜索
ctxvcs pull             # 一键打包当前全部状态（粘贴给任何 AI 助手用）
ctxvcs blame <主题名>    # 溯源
```

**给用 AI 助手工作的人**：运行一次 `ctxvcs install-skill`，之后在任何项目里对你的
AI 助手说「推送上下文」或「拉取团队上下文」即可，它会自动完成上面的流程。

## 四、管理员：处理冲突审核

网页顶栏的「审核」出现红色数字时，说明有推送和现有记录矛盾，在等你裁决：

1. 点进去，左边是**主干当前版本**，右边是**新推送的版本**，矛盾字段会高亮，
   系统会用一段话解释为什么判定为矛盾。
2. 看两边的**记录者、身份（人工/智能体）和时间**：一般来说，"有明确理由的近期改动"
   胜过"凭记忆的重复陈述"。
3. 点「采纳新版本」或「保留现有版本」（也可以「编辑」后合并两者）。
4. 裁决完成后自动生效，页面几秒内更新。裁决记录永久保存，之后在「溯源」里可以查到。

## 五、常见问题

| 现象 | 原因与处理 |
|---|---|
| push 后显示 dropped | 团队已有这条信息，正常，不用管 |
| 提示 needs review | 有矛盾等管理员裁决；其他条目会在裁决后一起生效 |
| 页面看起来没更新 | 等几秒刷新；仍不行请联系管理员重建页面（对数据无风险） |
| 登录失效 | 你在别处重新登录过，重新 `ctxvcs login` 即可 |
| 不确定某条信息可不可信 | 主题页点「溯源」——谁说的、什么身份、被质疑过吗，一目了然 |
